# Playwright 30-Day Roadmap

This roadmap is meant to keep practice simple: learn one topic, write tests for it, and record what you learned in [DAILY_NOTES.md](DAILY_NOTES.md).

## How To Use This Roadmap

1. Pick the topic for today.
2. Read only enough to understand the idea.
3. Write one or more tests for that topic in `tests/specs/`.
4. Refactor the test so it is clear and reusable.
5. Add a short note to [DAILY_NOTES.md](DAILY_NOTES.md).

## What To Practice Today

Today’s focus: basic actions and assertions.

Practice checklist:

- Navigate to a real site.
- Find elements with a good locator.
- Click, fill, or press a key.
- Verify text, title, URL, or visibility.
- Keep each test focused on one behavior.

Recommended practice file:

- [tests/specs/01-basic-actions-&-assertions.ts](tests/specs/01-basic-actions-&-assertions.ts)

## 30-Day Plan

- Day 1: Navigation, locators, basic assertions
- Day 2: Locator strategy
- Day 3: Actions
- Day 4: Assertions deeply
- Day 5: Waiting behavior and auto-wait
- Day 6: Multiple small tests on one page

- Day 8: Forms and validation

### Week 3: Test Architecture

- Day 15: Page Object Model properly

- Day 17: Fixtures and dependency injection style testing
- Day 18: Test data and TypeScript interfaces
- Day 19: Configuration, projects, timeouts, debug mode
- Day 20: Authentication and storage state
- Day 21: Refactor one old test into a cleaner architecture

### Week 4: Professional Practice

- Day 22: API testing basics
- Day 23: Network interception and mocking
- Day 24: Parallel runs, isolation, and flaky test prevention
- Day 25: Tracing, debugging, screenshots, and video

- Day 28: End-to-end flow on a realistic app
- Day 29: Small suite with pages, fixtures, and types
- Day 30: Final capstone and review

## Practice Rule

Do not try to learn many topics in one day. One topic, one practice session, one note.
